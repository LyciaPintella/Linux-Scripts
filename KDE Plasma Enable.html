Copilot said

The error you’re seeing means your kernel package linux-image-6.17.0-22-generic is broken, and until it is repaired or removed, APT cannot install KDE Plasma or any other package. This is a classic dpkg‑level failure, not a KDE issue.

Below is the exact sequence that reliably fixes this kind of kernel‑image breakage on Ubuntu.

1. Identify the broken kernel package

Your error explicitly names:

  • linux-image-6.17.0-22-generic

This kernel belongs to Ubuntu 25.10 (Questing Quokka), so if you're on an earlier Ubuntu release, this package may have been pulled from a proposed or mismatched repo — a common cause of dpkg failures.

2. Repair dpkg’s state

Run the dpkg reconfiguration step first:

bash
sudo dpkg --configure -a

This is the standard first fix for dpkg error code 1.

3. Force‑fix broken dependencies

Then run:

bash
sudo apt-get install -f

This forces APT to repair or complete partially installed packages.

4. If the kernel package is still broken, remove it

If steps 2–3 still leave the kernel image in a broken state, remove the problematic version:

bash
sudo apt-get purge linux-image-6.17.0-22-generic linux-modules-6.17.0-22-generic

This is safe as long as you still have another working kernel installed (Ubuntu always keeps at least one older kernel).

5. Update your package lists and reinstall a correct kernel

If your system is on Ubuntu 24.04, 24.10, or 25.04, you should not be using the 25.10 kernel. After purging, run:

bash
sudo apt update
sudo apt install --reinstall linux-image-generic linux-headers-generic

This ensures your kernel matches your Ubuntu release.

6. Now install KDE Plasma

Once the kernel is fixed and dpkg is clean:

bash
sudo apt install kde-plasma-desktop

It should now install without errors.

You’ve uncovered the real root of the failure:

zz-update-grub exited with return code 127 dpkg: error processing linux-image-6.17.0-22-generic (--configure)

This means the post-install script for the kernel is crashing, not KDE, not APT. Exit code 127 = “command not found”, which almost always means:

  • /usr/sbin/update-grub is missing, corrupted, or not executable

  • /etc/kernel/postinst.d/zz-update-grub is broken or pointing to a missing binary

  • The kernel package itself is mismatched with your Ubuntu release

Because this is a kernel postinst failure, APT cannot continue installing anything until this is fixed or removed.

Let’s walk through the exact sequence that fixes this class of error on Ubuntu.

1. Check whether update-grub is missing

Run:

bash
ls -l /usr/sbin/update-grub

If it says No such file or directory, then your GRUB package is broken.

Fix it:

bash
sudo apt-get install --reinstall grub-common grub2-common grub-pc-bin grub-pc

Then retry:

bash
sudo dpkg --configure -a

If update-grub does exist, continue.

2. Inspect the failing script

Run:

bash
ls -l /etc/kernel/postinst.d/zz-update-grub

If it is 0 bytes, not executable, or has a broken shebang, fix it:

bash
sudo chmod +x /etc/kernel/postinst.d/zz-update-grub

If it looks corrupted, reinstall GRUB again:

bash
sudo apt-get install --reinstall grub-common

3. If the kernel package itself is mismatched, purge it

Your kernel version:

6.17.0-22-generic

This kernel belongs to Ubuntu 25.10, not any stable release. If you’re on 24.04, 24.10, or 25.04, this kernel will always fail.

Purge it cleanly:

bash
sudo dpkg --remove --force-remove-reinstreq linux-image-6.17.0-22-generic

Then clean up:

bash
sudo apt-get install -f

4. Reinstall the correct kernel for your Ubuntu release

This ensures your system has a valid kernel and valid postinst scripts:

sudo apt install --reinstall linux-image-generic linux-headers-generic

5. Now repair dpkg

If this completes without errors, your system is healthy again.

Switching your display manager to SDDM is straightforward, and since you already have KDE installed, your system is basically ready for it. The only thing you need to do is install SDDM (if it isn’t already) and tell Ubuntu to use it instead of GDM.

Here’s the clean, reliable way to do it.

1. Install SDDM (if not already installed)

bash
sudo apt install sddm

If it’s already installed, APT will tell you.

2. Set SDDM as the default display manager

Run:

bash
sudo dpkg-reconfigure sddm

A dialog will appear with a list of display managers:

  • gdm3 (Ubuntu’s default)

  • sddm (KDE’s display manager)

  • possibly lightdm if installed

Use the arrow keys to select sddm, then press Enter.

This updates:

Code
/etc/X11/default-display-manager

and ensures SDDM starts on boot.

3. Reboot

bash
sudo reboot

When the system comes back up, you should see the KDE SDDM login screen instead of Ubuntu’s GDM.

4. (Optional) Apply a KDE theme to SDDM

Once logged into KDE:

  1. Open System Settings

  2. Go to Startup and Shutdown → Login Screen (SDDM)

  3. Choose a theme you like

  4. Click Apply

KDE has some gorgeous SDDM themes, and this is where you control them.

5. Verify SDDM is active

Run:

bash
cat /etc/X11/default-display-manager
